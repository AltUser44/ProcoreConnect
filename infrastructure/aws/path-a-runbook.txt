PATH A — "portfolio lean" deploy in us-east-1 (one EC2 + RDS + ECR; no ALB)

Prereqs
- You completed: aws configure --profile procoreconnect, region us-east-1, sts get-caller-identity works
- This repo: infrastructure/aws/ scripts

0) Do NOT commit secrets: never commit .pem, .env with DB passwords, or AWS access keys to git.
   (Root .gitignore ignores *.pem and .env files.)

0b) If run-instances fails on "--user-data" with Errno 22: your repo is likely under
   OneDrive; the script now copies user-data to %TEMP% and uses fileb:// (fixed in
   a280205+). Pull latest path-a-01-ec2.ps1 and re-run.

1) Create the EC2 app box (t3.micro + Docker) + Elastic IP
   Open PowerShell:
   cd C:\path\to\ProcoreConnect\infrastructure\aws
   Set-ExecutionPolicy -Scope CurrentUser -RemoteSigned
   .\path-a-01-ec2.ps1
   - Script creates an EC2 key pair and writes <name>.pem next to the script.
   - Tighten Windows permissions on the PEM (if storing locally):
     icacls ".\procoreconnect-procore-key.pem" /inheritance:r
     icacls ".\procoreconnect-procore-key.pem" /grant:r "%USERNAME%:R"
   - SSH: easiest from WSL, or use a Windows OpenSSH build:
     ssh -i procoreconnect-procore-key.pem ec2-user@<EIP from script output>
   - Wait 2-5 min for user-data, then: docker --version, docker compose version, tail /var/log/procoreconnect-bootstrap.log

2) RDS PostgreSQL (Free Tier db.t3.micro) — *next after EC2 is up*
   cd C:\path\to\ProcoreConnect\infrastructure\aws
   .\path-a-02-rds.ps1
   - Subnet group: two subnets in different AZs in the default VPC
   - procoreconnect-rds-sg: TCP 5432 from procoreconnect-app-sg only; RDS is not public
   - First run: writes master password to .rds-creds.local (gitignored). Or:
     $sec = Read-Host "Password" -AsSecureString; .\path-a-02-rds.ps1 -MasterPassword $sec
   - Set on the app host: DATABASE_HOST, DATABASE_PORT, DATABASE_USERNAME, PROCORECONNECT_DATABASE_PASSWORD
   - Production DB / user names match procoreconnect/config/database.yml (database procoreconnect_production)
   - If path-a-02-rds.ps1 is empty or PowerShell reports parse errors after a bad save, restore from git
     (from repo root): git checkout HEAD -- infrastructure/aws/path-a-02-rds.ps1

3) ECR: create repos + how to build/push
   .\path-a-03-ecr.ps1
   - From repo root, run the printed docker build / ecr get-login-password / tag / push commands
   - Path A EC2 must be able to pull: attach an instance profile to the instance with
     AmazonEC2ContainerRegistryReadOnly (or equivalent ECR read policy) — use IAM in the console, then
     re-boot or re-login; do not long-term store IAM user keys on the host.

4) Frontend static: S3 + CloudFront (or serve via nginx on EC2 to keep phase 1 minimal)
   - VITE_API_BASE_URL to your public https API URL when you add TLS

5) TLS
   - Path A with no ALB: put nginx + Let's Encrypt (certbot) on EC2, or front with CloudFront+ACM later.

6) Teardown
   - Terminate the EC2 instance, release the Elastic IP (small charge if you keep an *unassigned* EIP), delete the RDS, empty/delete S3 buckets, delete the key pair in EC2, remove security groups and VPC artifacts you created
